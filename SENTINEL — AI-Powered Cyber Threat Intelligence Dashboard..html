CONCEPT DEFINITIONS

1. SIEM (Security Info & Event Management)

Software platforms that aggregate, analyze, and map security telemetry from firewalls, servers, and endpoint security agents, transforming raw telemetry into readable, structured threat incidents.

2. MITRE ATT&CK Framework

A globally-accessible knowledge base of adversary tactics, techniques, and procedures (TTPs) based on real-world threat monitoring logs. It provides standard reference classifications for defensive design auditing.

3. IOCs (Indicators of Compromise)

Forensic artifacts observed on networks or hosts (e.g., known malicious domain names, file hashes, IPs) that verify a system compromise has taken place.

4. CVSS (Common Vulnerability Scoring System)

A standardized numerical rating scale (0 to 10) expressing the relative severity and impact mechanics of identified security vulnerabilities, with values of 9.0+ categorized as Critical threats.

5. MTTD & MTTR (Detection vs. Resolution)

Mean Time to Detect (MTTD) tracks the temporal speed with which security anomalies are identified. Mean Time to Respond (MTTR) gauges the velocity of operational security actions deployed to contain threats.
